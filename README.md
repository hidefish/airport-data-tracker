## 📦 部署步骤（5分钟）

### 1. 在 GitHub 创建仓库

```bash
# 仓库名: airport-data-tracker
# 访问: https://github.com/new
```

### 2. 克隆仓库到本地

```bash
git clone https://github.com/hidefish/airport-data-tracker.git
cd airport-data-tracker
```

### 3. 复制文件到仓库

将以下文件复制到对应位置：

```
airport-data-tracker/
├── .github/
│   └── workflows/
│       └── monthly_update.yml           # GitHub Actions 配置
├── data/
│   └── airports_data.json               # 初始数据
├── scripts/
│   └── fetch_multiyear_data.py          # 数据采集脚本
├── index.html                           # 可视化页面（重命名）
└── README.md                            # 说明文档
```

**文件映射**:
- `monthly_update.yml` → `.github/workflows/monthly_update.yml`
- `fetch_multiyear_data.py` → `scripts/fetch_multiyear_data.py`
- `top50_dashboard_v2.html` → `index.html`（重命名为首页）
- `data/airports_data.json` → `data/airports_data.json`

### 4. 推送到 GitHub

```bash
git add .
git commit -m "初始化机场数据追踪系统 v2.0"
git push origin main
```

### 5. 启用 GitHub Pages

1. 进入仓库 **Settings** → **Pages**
2. **Source** 选择 `main` 分支
3. 保存

等待 1-2 分钟后访问:
```
https://hidefish.github.io/airport-data-tracker/
```

### 6. 验证自动更新

1. 进入 **Actions** 标签
2. 点击 **每月更新机场数据**
3. 点击 **Run workflow** → 手动触发测试
4. 查看运行日志

## 🔧 数据校核机制

系统执行 **5 层独立校核**，符合"数据类任务铁律"：

### 1. 单条记录校核
```python
✅ 必填字段完整性
✅ 客流量合理性（15-150M）
✅ 增长率合理性（<500%）
✅ IATA 代码格式（3字母大写）
✅ 排名范围（1-100）
```

### 2. 数据集完整性
```python
✅ 数量检查（≥30 条）
✅ 排名连续性（无缺失/重复）
✅ Top 10 平均值合理性
```

### 3. 年度对比校核
```python
✅ 同比增长率逻辑性
✅ 排名变化合理性
✅ 区域分布稳定性
```

### 4. 数据来源标注
```json
{
  "metadata": {
    "sources": [
      {"name": "Wikipedia", "url": "...", "status": "success"},
      {"name": "CAAC", "url": "...", "status": "success"}
    ]
  }
}
```

### 5. 校核失败处理
```
❌ 校核失败 → 中止更新
⚠️  保存标记为 "validation_failed"
📧 GitHub Actions 生成错误报告
```

## 🎨 可视化功能

### 三种视图模式:

1. **柱状图** - 横向柱状图，按客运量排序
   ```
   用途：对比各机场客流规模
   颜色：按地区区分
   ```

2. **气泡图** - X轴排名，Y轴客流量，**气泡大小=增长率**
   ```
   用途：识别高增长机场
   特点：增长越快，气泡越大
   ```

3. **排名变化** - 散点图叠加 2024/2025/2026 三年数据
   ```
   用途：追踪排名变化趋势
   特点：三种颜色区分年份
   ```
